# CACAO FLOW

**Infraestructura de pre-underwriting agrícola para financiamiento pre-cosecha.**

CACAO FLOW convierte evidencia dispersa de un lote agrícola en un caso financiero **legible, auditable y revisable**.

---

## Resumen

El acceso a financiamiento agrícola pre-cosecha sigue siendo lento, manual y fragmentado.  
La información relevante para evaluar un caso —documentos, ubicación, evidencia operativa, proyección de rendimiento— suele llegar desordenada y sin una estructura clara para underwriting.

CACAO FLOW resuelve esa fricción transformando un lote en un flujo de evaluación claro:

**Lote → Evidencia → Score → VPC → Review financiera**

---

## Problema

Hoy el problema no es solo la falta de capital.

El problema es que **no existe una unidad financiera confiable, comparable y auditable** para evaluar un lote antes del desembolso.

### Esto genera tres fricciones principales:
- evidencia desordenada;
- análisis de riesgo poco transparente;
- baja estandarización para revisar y decidir.

---

## Nuestra solución

CACAO FLOW es una plataforma de **pre-underwriting agrícola** que transforma un lote en un **VPC (Verified Pre-Harvest Claim)**.

Un VPC resume:

- identidad del lote;
- evidencia asociada;
- score interpretable;
- rendimiento esperado ajustado;
- monto elegible;
- estado del caso;
- referencia verificable on-chain.

---

## Cómo funciona

| Etapa | Qué ocurre |
|---|---|
| Originación | El productor o cooperativa registra el lote |
| Evidencia | Se cargan documentos y datos operativos |
| Scoring | El sistema genera un score con breakdown visible |
| VPC | Se emite un claim auditable del lote |
| Review | Un financiador revisa y pre-aprueba el caso |
| Trazabilidad | Eventos clave se registran en Arbitrum |

---

## Qué hace distinto al producto

CACAO FLOW no intenta reemplazar bancos, aseguradoras ni toda la operación agrícola.

Su foco es una capa concreta:

**estructurar evidencia + estandarizar evaluación + crear un objeto revisable**

Eso convierte una demo de “agri-finance + blockchain” en una arquitectura de producto mucho más creíble.

---

## Rol de Arbitrum

Usamos Arbitrum como capa de **auditabilidad verificable**, no como sustituto de la operación off-chain.

### On-chain
- hash o fingerprint del VPC;
- timestamp de emisión;
- cambios críticos de estado;
- referencia verificable del claim.

### Off-chain
- documentos pesados;
- formularios completos;
- lógica de scoring;
- operación diaria;
- almacenamiento principal.

### Por qué esta arquitectura
Porque en productos financieros reales, el análisis y la operación viven off-chain, mientras que la blockchain aporta integridad, trazabilidad y prueba de existencia.
